# Prompt Fungineer v2: Distilling Claude's Prompt Engineering Into a Local 4B Model

## Background

In April 2023, I fine-tuned GPT-2 Medium (355M parameters) on image prompts to create the original [Prompt Fungineer](https://huggingface.co/treadon/prompt-fungineer-355M). It takes a simple text input like "a cat on a mountain" and expands it into a detailed, production-ready prompt for image generators (Midjourney, Stable Diffusion, DALL-E, Flux, etc.).

The 355M model works, but it's limited. It often produces generic expansions, sometimes hallucinates nonsensical photographer names, and lacks the nuance of a truly skilled prompt engineer. Three years later, the landscape has changed dramatically:

- Open-source models at 4B parameters now match what 175B models did in 2023
- Distillation from frontier models (Claude, GPT-4) into small models is a proven technique
- LoRA and full fine-tuning on Apple Silicon is practical and fast
- Image generators have evolved — they respond to different vocabulary and techniques than they did in 2023

**Can we build a dramatically better prompt fungineer by distilling Claude's prompt engineering ability into a small, local model?**

## Hypothesis

A full fine-tune of **Qwen3-0.6B** (600M params, 2025 architecture) on 9,400 high-quality prompt pairs generated by Claude will produce a prompt fungineer that dramatically outperforms the original GPT-2 355M — despite being only 1.7x larger. The improvement comes from:

1. **Better architecture** (GQA attention, RoPE, 151K vocab vs GPT-2's 50K)
2. **Better pretraining** (multilingual, code, reasoning vs English web text)
3. **Better training data** (9,400 Claude-generated pairs vs web-scraped prompts)

Specifically, we predict:
1. **More creative and varied expansions** than the GPT-2 355M original
2. **Better prompts that produce better images** when fed to modern image generators
3. **More consistent handling of diverse inputs** (scenes, objects, abstract concepts, emotions)
4. **Faster inference** on the same hardware (modern architecture optimizations)

## Research Questions

1. **Does architecture generation matter more than scale?** Comparing GPT-2 355M (2019 architecture) vs Qwen3-0.6B (2025 architecture) — same size class, 6 years of architectural progress.
2. **Does training data quality matter more than quantity?** The original was trained on scraped web data (large, noisy). The new model trains on 9,400 Claude-generated pairs (small, high quality). Which matters more?
3. **How close can a 600M model get to its Claude teacher?** Distillation ceiling — what's the quality gap between teacher and student?
4. **How do 2023 prompts compare to 2026 prompts?** Image generators have evolved. Do prompts need to evolve with them?
5. **Can a tiny model learn "taste"?** The original fungineer produces generic expansions. Can a better-trained small model learn the difference between a good prompt and a great one?

## Method

### 1. Dataset Generation (Teacher: Claude Opus 4.6)

Generate 20,000 prompt pairs using Claude:

**Input format:** A simple, short image description (2-10 words)
**Output format:** A detailed, production-ready image prompt (50-200 words)

The dataset will cover diverse categories:
- **Landscapes & nature** (2,500 pairs)
- **Portraits & people** (2,500 pairs)
- **Architecture & urban** (2,000 pairs)
- **Animals & wildlife** (2,000 pairs)
- **Food & still life** (1,500 pairs)
- **Abstract & artistic** (2,000 pairs)
- **Sci-fi & fantasy** (2,000 pairs)
- **Historical & vintage** (1,500 pairs)
- **Macro & close-up** (1,500 pairs)
- **Vehicles & technology** (1,000 pairs)
- **Emotions & concepts** (1,500 pairs)

Each prompt expansion includes:
- **Scene description** — what's in the image, with specific details
- **Lighting** — time of day, direction, quality, color temperature
- **Composition** — camera angle, framing, depth of field
- **Atmosphere/mood** — emotional tone, color palette
- **Technical** — camera model, lens, shutter speed, film stock (when appropriate)
- **Style reference** — artistic movement, visual aesthetic (not specific artist names)

Quality controls:
- No copyrighted character names or real artist names
- No NSFW content
- Varied sentence structures (not all starting with "A breathtaking image of...")
- Mix of photographic and artistic/painterly styles
- Range of prompt lengths (some concise, some detailed)

### 2. Model Training (Student: Qwen3.5-4B)

**Base model:** Qwen/Qwen3.5-4B
**Method:** Full fine-tune (all parameters unfrozen)
**Hardware:** Apple M4 Pro, 64GB unified memory, MPS backend

Training configuration:
- **Epochs:** 2-3 (early stopping on val loss)
- **Learning rate:** 1e-5 with cosine schedule + warmup
- **Batch size:** 4 (with gradient accumulation to effective batch 16)
- **Max sequence length:** 512 tokens
- **Optimizer:** AdamW, weight decay 0.01
- **Precision:** BF16
- **Val split:** 5% (1,000 pairs held out)

Expected training time: 4-8 hours on M4 Pro.

Memory budget:
- Model weights (BF16): ~8 GB
- Gradients: ~8 GB
- Optimizer states: ~16 GB
- Activations: ~4-8 GB
- Total: ~36-40 GB (fits in 64 GB)

### 3. Evaluation

**Quantitative:**
- Generate prompts for 100 held-out simple inputs
- Score expanded prompts on: detail richness, coherence, creativity, technical accuracy
- Use Claude as a judge (0-10 per dimension)
- Compare: GPT-2 355M vs Qwen 4B (ours) vs raw Claude (teacher ceiling)

**Qualitative:**
- Run all three models' prompts through an image generator (Z-Image-Turbo, which we already have locally)
- Visual comparison: do better prompts → better images?
- Blind test: show pairs of images to people, ask which prompt produced the better image

**Speed:**
- Tokens per second on M4 Pro for each model
- Time to generate a single prompt expansion

### 4. Deliverables

1. **HuggingFace Model:** `treadon/prompt-fungineer-v2-4B`
2. **HuggingFace Dataset:** `treadon/prompt-fungineer-v2-training-data` (20K Claude-generated pairs)
3. **HuggingFace Space:** Upload a simple prompt, get expansions from v1 (355M), v2 (4B), and optionally raw Claude
4. **GitHub Repo:** Training code, evaluation scripts, results
5. **Blog Post:** Interactive comparison (toggle between v1/v2/Claude outputs + generated images)
6. **Tweet Thread:** Before/after comparisons with generated images

## Success Criteria

- v2 scores **>7/10 average** on Claude-judged quality (vs ~5/10 for v1)
- v2 prompts produce **visibly better images** than v1 prompts in blind comparison
- Model runs at **>20 tokens/second** on M4 Pro (interactive speed)
- Total model size **<10 GB** (downloadable, local-friendly)

## Risks

- **Overfitting:** 20K samples on 4B parameters — mitigated by low learning rate, few epochs, early stopping
- **Mode collapse:** Model might learn a narrow style — mitigated by diverse training categories
- **MPS training issues:** Apple Silicon training can be flaky — fallback: use CPU (slower but stable)
- **Claude as judge bias:** Claude might prefer its own style — mitigated by also using image generation as an objective measure

## Why This Matters

This experiment tests a broader question: **can you distill a specific skill from a frontier AI model into a small, local model that anyone can run?**

If it works for prompt engineering, the same approach applies to:
- Code review (distill Claude's code feedback into a local model)
- Writing editing (distill style suggestions)
- Data analysis (distill analytical reasoning)
- Domain-specific Q&A (distill expert knowledge)

The banger scorer showed we can evaluate AI output quality. The prompt fungineer shows we can improve AI *input* quality. Together: better inputs → better outputs → better evaluation → better inputs. The optimization loop.
